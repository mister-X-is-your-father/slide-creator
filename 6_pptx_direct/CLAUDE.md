# 6_pptx_direct - .pptx直接生成

## 概要

Claude Max/Team/Enterpriseプランのファイル作成機能で直接PowerPoint生成。
追加ツール不要、即座にPowerPoint形式で出力。

## ディレクトリ構成

```
6_pptx_direct/
├── 1_input/           # 下書き・骨子
├── 2_output/          # 生成されたPPTX
├── prompts/           # プロンプトテンプレート
└── CLAUDE.md          # 本ファイル
```

## 前提条件

- Claude Max、Team、または Enterpriseプラン
- Claude.ai Web版（ファイル作成機能が必要）

## ワークフロー

```
1. 1_input/ に骨子を準備
    ↓
2. prompts/ のテンプレートを参考にプロンプト作成
    ↓
3. Claude.ai でプロンプト実行
    ↓
4. 生成された .pptx をダウンロード
    ↓
5. 2_output/ に保存
```

## 推奨プロンプト構成

### 1. 事前に骨子を作成

別のAI（Gemini等）やメモで先に構成を整理してからClaudeに依頼すると品質向上。

### 2. 3層構造を指定

各スライドで:
- タイトル: 何について
- リード文: 結論（1-2行）
- ボディ: 根拠・詳細

### 3. ビジュアルの指示

- 無闘なグラフ化を避ける
- わかりやすさ優先
- シンプルな配色

## 制限事項

- **有料プラン必須**: Max/Team/Enterprise
- **部分修正が困難**: 修正は全体再生成が必要
- **5時間セッション制限**: インタラクティブな修正を繰り返すと制限に達する可能性
- **最終調整は手動**: レイアウト微調整はPowerPointで行う

## メリット

- 追加ツール不要
- 即座にPowerPoint形式
- 叩き台として十分なクオリティ
- 従来数時間→約5分で完了

## いつ使うか

- 急ぎの資料作成
- 叩き台が欲しい時
- PowerPoint形式が必須の場合
- Marp等の学習コストを避けたい場合

## いつ使わないか

- 頻繁な修正が予想される場合 → Marpアプローチを推奨
- デザインの細かい調整が必要 → 2_marp_design_systemを推奨
- バージョン管理したい場合 → Markdown系アプローチを推奨
